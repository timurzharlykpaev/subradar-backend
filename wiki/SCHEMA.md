# Wiki Schema — SubRadar Backend

## Назначение

LLM-ориентированная база знаний для кодовой базы subradar-backend.
Используется Claude Code и другими LLM-агентами для быстрого поиска архитектурных решений, контрактов API, бизнес-логики.

## Структура директорий

```
wiki/
  SCHEMA.md          # Этот файл — правила ведения wiki
  index.md           # Индекс всех страниц (по категориям)
  log.md             # Журнал изменений wiki
  pages/             # Основные страницы знаний
    overview.md
    architecture.md
    subscriptions-module.md
    auth-module.md
    billing-module.md
    analytics-module.md
    ai-module.md
    fx-module.md
    users-module.md
    database.md
    notifications-module.md
    deploy.md
    api-contracts.md
    known-issues.md
  sources/            # Необработанные заметки, дампы, временные файлы
```

## Формат страницы

Каждая страница в `pages/` начинается с YAML frontmatter:

```yaml
---
title: Название страницы
tags: [module, subscriptions, crud]
sources:
  - src/subscriptions/subscriptions.service.ts
  - src/subscriptions/entities/subscription.entity.ts
updated: 2026-04-16
---
```

### Обязательные поля frontmatter

| Поле | Описание |
|------|---------|
| `title` | Человекочитаемое название |
| `tags` | Массив тегов для поиска |
| `sources` | Исходные файлы, на которых базируется страница |
| `updated` | Дата последнего обновления (ISO формат) |

## Именование файлов

- Формат: `kebab-case.md`
- Суффикс `-module` для страниц NestJS модулей
- Без пробелов, только латиница и дефисы

## Кросс-ссылки

Используйте `[[page-name]]` для ссылок между страницами:

```markdown
Подробности конвертации см. в [[fx-module]].
Пользовательские настройки описаны в [[users-module]].
```

## Рабочие процессы

### Ingest (добавление информации)

1. Прочитать исходные файлы
2. Создать/обновить страницу в `pages/`
3. Обновить frontmatter `sources` и `updated`
4. Обновить `index.md` если добавлена новая страница
5. Добавить запись в `log.md`

### Query (поиск информации)

1. Открыть `index.md` — найти нужную категорию
2. Перейти на страницу по ссылке `[[page-name]]`
3. При необходимости — пройти по кросс-ссылкам
4. Если информации нет — прочитать `sources` файлы из frontmatter

### Lint (проверка актуальности)

1. Проверить `updated` дату — если старше 30 дней, пометить для обновления
2. Проверить `sources` — убедиться что файлы существуют
3. Проверить кросс-ссылки — убедиться что целевые страницы существуют
4. Проверить frontmatter — все обязательные поля заполнены

## Правила обновления

- При изменении кода, затрагивающего задокументированный модуль, обновить соответствующую wiki-страницу
- Не удалять старые записи из `log.md` — только добавлять новые сверху
- В `index.md` поддерживать алфавитный порядок внутри категорий
- Содержание должно быть на русском языке
